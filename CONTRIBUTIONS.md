# Внески команди

## Вероніка
- Реалізація Products API (створення, перегляд, пошук товарів).
- Бізнес-логіка управління товарами:
  - оновлення цін з використанням optimistic locking;
  - ведення історії змін цін;
  - логічне видалення товарів (soft delete).
- Реалізація аналітичного SQL-запиту:
  - визначення топ-3 товарів у кожній категорії.
- Реалізація та підтримка частини Orders API через інтеграційне тестування.
- Написання інтеграційних тестів:
  - Products API (`products.test.ts`);
  - Orders API (`orders.test.ts`).
- Підготовка та оформлення документації:
  - опис запуску додатку;
  - опис запуску тестів;
  - приклади API-запитів;
  - опис структури проєкту в README.md.
- Забезпечення якості та стабільності бізнес-процесів шляхом тестування.

## Юля
- Ініціалізація проєкту та базової архітектури серверної частини.
- Проєктування та реалізація схеми бази даних (`schema.prisma`).
- Виконання та супровід міграцій бази даних (Prisma Migrate).
- Налаштування Docker та PostgreSQL для локального середовища розробки.
- Реалізація Orders API:
  - створення замовлень з товарами;
  - транзакційна логіка та перевірка залишків;
  - оновлення статусів замовлень з optimistic locking.
- Реалізація Users API.
- Реалізація аналітичного SQL-запиту:
  - Customer Lifetime Value (LTV).
- Підготовка та оформлення документації:
  - документація схеми бази даних (`docs/schema.md`);
  - інструкції з налаштування проєкту;
  - опис архітектурних рішень.
- Merge pull request та фінальна інтеграція коду.
